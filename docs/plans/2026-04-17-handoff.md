## Problem 1 — admin password login fails on `allen` — RESOLVED BY REINSTALL

**Password**: `[^_}cr3{6$TPz!?{\9bA` (20 chars). Same password intended for all four.

**Root cause (strong suspect): keyboard layout mismatch.**
Install config had `keyboard_layout: us`. User is UK-based on a physical UK
keyboard. Password contains `\` (0x5c) — on a UK keyboard with a US-layout
system, the `#`-labelled physical key (second row, next to Enter) produces
`\`, so the password typed at the login prompt doesn't match the stored hash.

**Decision (2026-04-20): reinstall allen rather than patch the running system.**
All four configs updated to `keyboard_layout: gb` plus static IPs — see
"Current state" table below for the assigned IPs. Allen's stale ISO
`isos/built/allen-20260417.iso` is obsolete; rebuild before reflashing.

**Steps to reinstall allen:**
1. `uv run autoboot build allen` — produces a new dated ISO with `gb` layout
   and static IP `10.46.172.130/24`.
2. Flash to USB, boot allen from it. **This wipes the current allen install.**
3. Post-install, allen is reachable at `10.46.172.130` as `ansible` via the
   SSH key at `keys/ansible.pub`.

Betty/roxy/barry get the same `gb` + static-IP treatment via their updated
configs; rebuild each before flashing.
